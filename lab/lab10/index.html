Introduction to Exceptions

Exceptions allow us to try a chunk of code, and then catch any errors that might come up. If we do catch an exception, we can run an alternative set of instructions. This construct is very useful in many situations.

try:
    <try suite>
except Exception as e:
    <except suite>
else:
    <else suite>
finally:
    <finally suite>

Notice that we can catch the exception as e. This assigns the name e to the exception object. This can be helpful when we want to give extra information on what happened. For example, we can print(e) inside the except clause.

Also, we have an optional else case. The else suite is executed if the try suite finishes without any exceptions.

We also have an optional finally clause, which is always executed, whether or not an exception is thrown. We generally don't need to use the else and finally controls in this class.

When we write exception statements, we generally don't just use the word Exception as above. Rather, we figure out the specific type of exception that we want to handle, such as TypeError or ZeroDivisionError. To figure out which type of exception you are trying to handle, you can type purposely wrong things into the interpreter (such as 'hi' + 5 or 1 / 0) and see what kind of exception Python spits out.

Question 2: Safe use of higher order functions

When writing higher order functions that take functions as arguments, there is always the possibility that the function that is passed in may have an error that results in an exception. In this problem we will use exception handling to deal with that. The function safe_sum is passed a function and a sequence; it is to return the sum of the results of applyng the function to the elements in the sequence. However, for elements in the sequence where the function throws an exception, the missing value is added into the partial sum.

We have started the solution by writing a wrapper function that is used to catch the exception. Complete it.

def safe_sum(fun, seq, missing=0):
    """Return the sum of fun applied to elements in seq using missing as a replacement
    for those elements on which fun throws an exception

    >>> safe_sum(lambda x: x, [5, "terrible", 4, 3, "two", 1])
    13
    >>> safe_sum(lambda x: 1/x, [1, 2, 0, 3, None, "bad"])
    1.8333333333333333
    """
    def wrap(fun, x):

        "*** YOUR CODE HERE ***"
        

        try:
            return fun(x)
        except (ZeroDivisionError, TypeError, ValueError) :
            return missing

    psum = 0
    for x in seq:
        psum += wrap(fun, x)
    return psum

Question 3: Quidditch

It's time for the opening quidditch match of the season! We represent the various positions for players with the Player class and its subclasses. Every player begins with a base_energy level, but every position requires a different proportion of energy. Every player has also been playing for time minutes. Fill in the energy method for the Beater, Chaser, Seeker, and Keeper classes, according to their docstrings. In addition, fill in the __init__ method for the Chaser class.

class Player:
	def __init__(self, name, base_energy, time):
		"""
		Players begin with base_energy and have been 
		playing for time minutes.
		"""
		self.name = name
		self.base_energy = base_energy
		self.time = time

	def energy(self):
		return self.base_energy

class Beater(Player):
	role = "bludgers"

	def energy(self):
		"""
		Returns the amount of energy left after time minutes. 
		For Beaters, this is their base energy level divided by 
		the number of minutes.
		>>> fred = Beater("Fred Weasley", 640, 40)
		>>> fred.energy()
		16.0
		"""

		"*** YOUR CODE HERE ***"
		

		return self.base_energy / self.time

class Chaser(Player):
	role = "score"

	def __init__(self, name, base_energy, time, goals):
		"""
		Chasers have a name, score goals in time minutes, 
		and begin with base_energy.
		"""

		"*** YOUR CODE HERE ***"
		

		self.name = name
		self.base_energy = base_energy
		self.time = time
		self.goals = goals

	def energy(self):
		"""
		Returns the amount of energy left after time minutes.
		However, Chasers only lose energy after they score a 
		goal. When they score a goal, they lose 10% of their
		energy.
		>>> katie = Chaser("Katie Bell", 230, 20, 2)
		>>> katie.energy()
		186.3
		"""

		"*** YOUR CODE HERE ***"
		

		energy = self.base_energy
		for i in range(self.goals):
			energy = energy * 0.9
		return energy

class Seeker(Player):
	role = "snitch"
	energy_expended = 0.6

	def energy(self):
		"""
		Returns the amount of energy after time minutes.
		Seekers expend 60% of their energy in the given time.
		>>> harry = Seeker("Harry Potter", 700, 30)
		>>> harry.energy()
		280.0
		"""

		"*** YOUR CODE HERE ***"
		

		return self.base_energy * (1 - Seeker.energy_expended)

class Keeper(Player):
	role = "guard"
	energy_expended = 0.15

	def energy(self):
		"""
		Returns the amount of energy after time minutes.
		If less than 30 minutes have passed, then keepers do 
		not lose any energy. If 30 minutes or more have passed, 
		then keepers expend 15% of their energy for every full 
		15 minutes that pass.
		>>> oliver = Keeper("Oliver Wood", 380, 45)
		>>> oliver.energy()
		233.3675
		"""

		"*** YOUR CODE HERE ***"
		

		energy = self.base_energy
		if self.time < 30:
			return self.base_energy
		else:
			for i in range(self.time // 15):
				energy = energy * (1 - Keeper.energy_expended)
		return energy

Question 4: Mafia

Now, you want to play Mafia with a group of your friends. A Player can either be a MafiaMember, or a TownsPerson. Fill in the methods in the Game class according to the docstrings.

import random

def get_most_common_element(lst):
	return max(set(lst), key=lst.count)

class Player:
	def __init__(self, name):
		self.name = name
		self.active = True

class MafiaMember(Player):
    def __init__(self, name):
    	Player.__init__(self, name)

    def reveal_player_type(self):
    	print("You are a mafia member!")    	

class TownsPerson(Player):
    def __init__(self, name):
    	Player.__init__(self, name)    

    def reveal_player_type(self):
    	print("You are a towns person!")

class Game:
	def __init__(self, players):
		if len(players) < 3:
			raise Exception("Not enough players!")
		mafia_names = random.sample(players, 3)
		self.mafia_members = [MafiaMember(mafia_name) for mafia_name in mafia_names]
		self.townspeople = [TownsPerson(name) for name in players if name not in mafia_names]
		self.name_to_player = {}

		for mafia_mem in self.mafia_members:
			self.name_to_player[mafia_mem.name] = mafia_mem

		for towns_person in self.townspeople:
			self.name_to_player[townspeople.name] = towns_person

		for i in range(len(self.name_to_player)):
			print("Please type your name and your player type will be revealed.")
			name = input()
			self.name_to_player[name].reveal_player_type()

		self.state = "night"

	def play(self):
		"""
		While the game is still being played, make a move.
		"""
		while not self.check_if_end_of_game():
			self.make_move()

	def make_move(self):
		"""
		Every player votes. Then, if the state of the game is day, remove the 
		player with the most votes overall, and set the state to night. If 
		the state of the game is night, remove the player with the most 
		mafia votes, and set the state to day.
		"""
		print("It is " + self.state + " time.")

		votes = []
		mafia_votes = []

		for name, player in self.name_to_player.items():
			print('Player ' + name + ', please vote on a player to remove from the game.')
			vote = input()
			if self.state == "night" and isinstance(player, MafiaMember):
				mafia_votes.append(vote)
			votes.append(vote)

		if self.state == "day":
			majority_vote = get_most_common_element(votes)
			self.state = "night"
		elif self.state == "night":
			majority_vote = get_most_common_element(mafia_votes)
			self.state = "day"
		self.remove_player(majority_vote)

	def remove_player(self, player_to_remove):
		"""
		Set the player with the majority vote to inactive, and remove it from 
		its respective list of players.
		"""
		player = self.name_to_player[player_to_remove]
		player.active = False

		if player in self.mafia_members:
			self.mafia_members.remove(player)
		elif player in self.townspeople:
			self.townspeople.remove(player)

	def check_if_end_of_game(self):
		"""
		Returns True if the game is over, and False if it is not. The game is over when 
		there are no members of the mafia remaining, or if there are more members of 
		the mafia than townspeople.
		"""
		if len(mafia_members) == 0:
			return True
		elif len(mafia_members) > len(townspeople):
			return True
		else:
			return False
